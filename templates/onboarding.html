What This Is

Niles Heron (your colleague at Verse and Hook) uses a private GitHub repository as persistent memory for his AI workflow. Instead of re-explaining context at the start of every session, that context lives in a structured repo that Claude — and other AI tools — can load on demand.

The repo holds an AI interaction profile, a task dashboard and working context for active projects, deliverables and reference material organized by project, and connections to the tools you already use. By the end of this session you'll have your own version.

How to Use This Guide

Option A — Let Claude run the session (recommended)

Click "Copy guide for Claude" above, paste into a new Claude conversation, and say: "Load this and run the onboarding." Claude interviews you, builds your profile, and walks you through each step.

Option B — Follow it yourself

Read through the sections below and complete each step on your own. Ask Claude for help on individual steps as needed.

Step 1 — GitHub Setup

Create an account (if needed)

Go to github.com and sign up with your work email. Free tier includes private repos. Choose a professional username — firstnamelastname works well.

Create your repo

  1. From your GitHub dashboard, click "+" → "New repository"
  2. Name it [username]-ai-management
  3. Set to Private, check "Add a README file"
  4. Click "Create repository"

Set up the folder structure

GitHub doesn't store empty folders. Create each one via the web UI: "Add file" → "Create new file" → type profile/.gitkeep. The slash creates the folder. Repeat for each:

[username]-ai-management/
├── profile/
├── tasks/
├── deliverables/
├── reference/
├── context/
└── templates/

Step 2 — Build Your AI Profile

The profile tells Claude who you are and how to work with you. If loading this into Claude, it'll interview you and build the profile from your answers.

What the profile covers: professional context (role, org, projects, stakeholders), communication preferences (tone, voice, AI habits that annoy you), collaboration mode (thinking partner vs. executor), integrations, and notes for AI.

Keep a full version (profile-v1.0-full.md) and a condensed version (~40% of the length) for mobile sessions. Condensed keeps role, communication prefs, and collaboration mode — cuts the detailed project and stakeholder lists.

Step 3 — Connect Claude

Connect GitHub to Claude.ai

  1. Go to claude.ai → Settings → Connected Apps
  2. Find GitHub and click Connect
  3. Authorize via OAuth — select your [username]-ai-management repo
  4. Grant read and write access (write lets Claude commit back)
  5. Confirm it shows as active

Note: GitHub MCP may not work on iPad/iPhone even after authorization — a known limitation. The Gist fallback below handles this.

Set up userPreferences

Claude.ai → Settings → Profile. Paste the following, filling in your details:

Load my AI interaction profile before starting significant work.

Primary source (GitHub):
repo: [github-username]/[username]-ai-management
path: profile/profile-v1.0-full.md

If GitHub is unavailable (e.g., mobile):
Load condensed version from this URL:
[GIST URL — see next step]

Before significant co-work, load the full profile and confirm you've loaded it.

Create a Gist fallback

  1. Go to gist.github.com
  2. Create a new public Gist
  3. Paste in your condensed profile, name the file profile-condensed.md
  4. Save, click "Raw," copy that URL from the browser
  5. Paste it into your userPreferences block above

Test it

Open a new Claude conversation and say: "Load my profile." Claude should load from GitHub and confirm. If it fails, test the Gist URL directly.

Step 4 — Integrations

Connect only what you actively use. All via Claude.ai → Settings → Connected Apps.

Gmail

Create a Gmail label called claude_tasks — apply it to emails you want Claude to act on. In sessions: "Check my Gmail for the claude_tasks label."

Google Drive

Claude can search and read Drive files. Read-focused — doesn't write back to Drive.

Granola

Install Granola, let it run during meetings, connect via Connected Apps. At session start: "Check Granola for recent action items."

Slack

Create a private channel called #[firstname]-claude-inbox. Drop tasks and notes there throughout the day — Claude processes them at the start of your next session. Find the channel ID in the browser URL: app.slack.com/client/[workspace]/[CHANNEL-ID].

Vercel / v0

v0.dev generates working HTML and React components from a description. Vercel deploys them live. Claude generates the code, commits to your public repo, Vercel deploys, you get a URL.

  1. Create accounts at v0.dev and vercel.com (GitHub login for both)
  2. Connect your GitHub account to Vercel for one-click deploys
  3. Connect Vercel in Claude.ai → Connected Apps if available

Other platforms

Check Claude.ai Connected Apps for anything else you use — Google Calendar, Notion, Linear, Asana, Monday.com.

Step 5 — Public Repo for HTML Files (Optional)

Niles hosts lightweight pages at dev.nilesheron.com via GitHub Pages. Claude produces HTML, commits it, Pages serves it at a stable URL automatically.

  1. New public GitHub repo — name it [username]-pages, initialize with README
  2. Repo → Settings → Pages → "Deploy from a branch" → main → / root → Save
  3. Your URL: https://[username].github.io/[repo-name]/

Optional custom domain: DNS CNAME pointing your subdomain to [username].github.io, then set it in repo Settings → Pages → Custom domain. Enable HTTPS.

Add to your profile:

github:
  public_repo: "[username]/[public-repo-name]"
  public_url: "https://[your-url]/"

Keeping It Useful

  • At the end of meaningful sessions, ask Claude to commit a context note
  • New project → add reference/[project-name]/
  • Role or priorities shift → update the profile
  • Use your Slack inbox or Gmail label for quick captures throughout the day
