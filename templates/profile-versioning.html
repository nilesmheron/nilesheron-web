Why Profiles Go Stale

A profile written at the start of a new role won't reflect what you're actually doing six months in. Projects change. Priorities shift. You learn things about how you want AI to work with you that you couldn't articulate before. The profile needs to keep up.

A stale profile is worse than no profile in one specific way: it gives Claude confident but wrong context. It'll write in a tone you've moved past, reference projects that are done, or apply preferences you've since changed. You'll notice it as friction — outputs that feel slightly off, context that doesn't quite apply.

Signals That It's Time to Update

Update immediately when

  • Your role, title, or primary organization changes
  • A major project launches, ends, or pivots significantly
  • A key stakeholder relationship changes — new contact, departed colleague, shifted dynamic
  • You catch Claude repeatedly doing something you've corrected more than twice
  • You add a new tool integration (Granola, Slack, a new platform)

Update on a regular cadence (monthly or quarterly) for

  • Active projects list — add new ones, mark completed ones for removal
  • Current priorities — what you're focused on changes even if the role doesn't
  • Communication preferences — these evolve as you use AI more; things that annoyed you early may be less important now, and new patterns will emerge
  • Collaboration mode notes — how you want AI to work with you deepens over time

Signs the profile is working well (no update needed)

  • Claude's first draft on a new task is close enough to edit rather than rewrite
  • Tone feels like you without prompting
  • Claude flags the right things proactively — asks the right clarifying questions
  • You rarely need to re-explain context that should already be there

Version Numbering

Change type Version bump Example
Fix a typo, update a date, add one project Patch: 1.0 → 1.0.1 Added new client project
Update preferences, revise communication notes, refresh projects list Minor: 1.0 → 1.1 Tightened tone guidance, removed completed projects
Role change, significant rewrite of multiple sections, structural overhaul Major: 1.x → 2.0 New organization, new primary use cases, different collaboration style

The filename stays the same — profile-v1.0-full.md becomes profile-v1.1-full.md. Update your userPreferences in Claude settings to point to the new filename when you do a minor or major bump. For patch updates, you can keep the same filename and just update the version header inside the file.

Version header format

Every profile file should have this at the top:

version: "1.1"
last_updated: "2026-05-12"
changelog:
  - "1.1 (2026-05-12): Updated active projects, tightened tone guidance, added Granola integration"
  - "1.0 (2026-04-08): Initial profile"

What to Actually Change

Communication preferences — the highest-leverage section

This is where most of the value lives. After a few months of working with Claude, you'll have concrete examples of things that work and things that don't. Be specific. "Avoid overly smooth prose" is less useful than "Don't soften direct feedback into something that reads as diplomatic hedging." Real examples from your actual outputs are the best raw material for updating this section.

Ask Claude to help you articulate it: "Here's an output I liked. Here's one that missed. What's the difference?" It can identify patterns you haven't consciously named yet.

Active projects — keep it current

This should reflect what you're actually working on right now, not a historical record. Completed projects should come out. New ones go in with a brief description and the key stakeholders Claude needs to know. Projects that are paused but will resume can stay with a note.

Collaboration mode — update as trust builds

Early profiles tend toward more checking in and confirmation steps. As you develop a working rhythm with Claude, you'll find places where you want it to move faster and execute more directly. Update the collaboration mode notes to reflect where you've landed — what you want challenged, what you want executed without a lot of back-and-forth.

Stakeholders — prune and add

If you've added notes about someone who's no longer in the picture, remove them. If a new relationship has become important enough that Claude should know the context going in, add a brief entry. Keep it to relationships where the context actually changes how Claude should approach something — not just a contact list.

Updating the Condensed Version

When you update the full profile, update the condensed version too. The condensed version is what loads on mobile and in quick sessions — if it's out of sync with the full profile, you'll get inconsistent behavior depending on the device.

The condensed version should always have current role, current communication preferences, and current collaboration mode. Everything else is secondary. When in doubt: if it would meaningfully change how Claude works with you, it belongs in the condensed version.

Update the Gist too — that's what actually loads from the URL in your userPreferences fallback. Updating the file in GitHub without updating the Gist means the mobile fallback is still running the old version.

Doing an Audit with Claude

The fastest way to update a profile is to ask Claude to audit it. The interactive guide at the top of this page runs that session. What it does:

  1. Reads your current profile from GitHub
  2. Asks you a series of targeted questions: what's changed, what's felt off, what's been missing
  3. Flags sections that look stale based on your answers
  4. Proposes specific changes — you approve, modify, or reject each one
  5. Drafts the updated profile and commits it
  6. Updates the condensed version and reminds you to update the Gist

Plan for about 20-30 minutes for a minor update, longer for a major revision. The changelog entry at the top of the profile should summarize what changed — write it like a commit message, not a diary entry.

One Rule

Don't let the profile get more than three months behind reality. If you're using Claude regularly and haven't touched the profile in that long, something important has changed that isn't reflected. Set a calendar reminder if that's what it takes. The maintenance cost is low; the cost of a stale profile compounds quietly until you notice you've stopped trusting the outputs.
