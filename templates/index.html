Colleague AI Onboarding Guide

v1.0 — April 2026 — Verse and Hook

Niles Heron uses a private GitHub repository as persistent memory for his AI workflow. Instead of re-explaining context at the start of every session, that context lives in a structured repo that Claude (and other AI tools) can load on demand.

This guide builds your own version of that system — personalized to how you work, connected to the tools you use. Load it into Claude and it'll run you through the setup interactively. Or read through it here first.

What the repo holds

  • An AI interaction profile — who you are, how you work, what you need from AI
  • A task dashboard and working context for active projects
  • Deliverables, reference material, and notes organized by project
  • Integrations with the tools you already use

How to use this guide

Option A — Let Claude run the session (recommended)

Click "Copy guide for Claude" above, paste it into a new Claude conversation, and say:

"Load this and run the onboarding."

Claude will ask you questions, build your profile from your answers, and walk you through each step.

Option B — Follow it yourself

Read through the sections below and complete each step on your own. You can still ask Claude for help on specific steps.

Step 1 — GitHub Setup

Create an account (if needed)

Go to github.com and sign up with your work email. The free tier includes private repos. Choose a username that's professional — firstnamelastname works well.

Create your repo

  1. From your GitHub dashboard, click "+" → "New repository"
  2. Name it [username]-ai-management
  3. Set to Private
  4. Check "Add a README file"
  5. Click "Create repository"

Set up the folder structure

GitHub doesn't store empty folders. Create each one via the web UI: click "Add file" → "Create new file" and type a path like profile/.gitkeep — the slash creates the folder automatically.

[username]-ai-management/
├── profile/
│   ├── profile-v1.0-full.md
│   └── profile-v1.0-condensed.md
├── tasks/
│   ├── task_dashboard.md
│   └── task_context.md
├── deliverables/
├── reference/
│   └── [project-name]/
├── context/
└── templates/

Step 2 — Build Your AI Profile

The profile is the most important file in the repo. It tells Claude — and any other AI — who you are and how to work with you. If you're loading this into Claude, it'll interview you and build the profile from your answers. If you're doing it yourself, the template below is your starting point.

What the profile covers

  • Professional context — role, org, active projects, key stakeholders
  • Communication preferences — tone, voice, things AI does that annoy you
  • Collaboration mode — how you want Claude to work with you (thinking partner vs. executor)
  • Integrations — which tools are connected and how
  • Notes for AI — anything else it should know

Two versions

Keep a full version (profile-v1.0-full.md) with all the detail, and a condensed version (~40% of the length) for mobile sessions where loading is slower. The condensed version keeps role, communication prefs, and collaboration mode — and cuts the detailed project and stakeholder lists.

Step 3 — Connect Claude

Connect GitHub to Claude.ai

  1. Go to claude.ai → Settings → Connected Apps
  2. Find GitHub and click Connect
  3. Authorize via OAuth — select your [username]-ai-management repo
  4. Grant read and write access (write lets Claude commit back)
  5. Confirm it shows as active

Note: GitHub MCP may not work on iPad/iPhone even after authorization — a known limitation. The Gist fallback below handles this.

Set up userPreferences

Go to Claude.ai → Settings → Profile and paste the following (filling in your details):

Load my AI interaction profile before starting significant work.

Primary source (GitHub):
repo: [github-username]/[username]-ai-management
path: profile/profile-v1.0-full.md

If GitHub is unavailable (e.g., mobile):
Load condensed version from this URL:
[GIST URL — see next step]

Before significant co-work, load the full profile and confirm you've loaded it.

Create a Gist fallback

  1. Go to gist.github.com
  2. Create a new public Gist
  3. Paste in your condensed profile
  4. Name the file profile-condensed.md
  5. Save, click "Raw," and copy the URL from the browser
  6. Paste that URL into your userPreferences block above

Test it

Open a new Claude conversation and say: "Load my profile." Claude should load the file from GitHub and confirm. If it fails, test the Gist URL directly.

Step 4 — Integrations

Connect only what you actively use. For each integration: Claude.ai → Settings → Connected Apps.

Gmail

Claude can search, read, and draft emails. For task management, create a Gmail label called claude_tasks — apply it to emails you want Claude to act on. Then in your session: "Check my Gmail for the claude_tasks label."

Google Drive

Claude can search and read Drive files — useful for pulling in docs, decks, and reference material. Read-focused; Claude does not write back to Drive.

Granola

Install Granola and let it run during meetings. Connect it via Connected Apps. At the start of sessions: "Check Granola for recent action items."

Slack

Create a private channel called #[firstname]-claude-inbox. Drop tasks, notes, and links there throughout the day. Claude processes them at the start of your next session. Find the channel ID in the browser URL: app.slack.com/client/[workspace]/[CHANNEL-ID].

Vercel / v0

v0.dev generates working HTML and React components from a description. Vercel deploys them live. Together: Claude generates the code, commits it to your public repo, Vercel deploys automatically, you get a URL to share.

  1. Create accounts at v0.dev and vercel.com (GitHub login works for both)
  2. Connect your GitHub account to Vercel for one-click deploys
  3. Connect Vercel in Claude.ai → Connected Apps if available

Step 5 — Public Repo for HTML Files

Optional but useful. Niles hosts lightweight pages at dev.nilesheron.com via GitHub Pages — Claude produces HTML, commits it, GitHub Pages serves it at a stable URL.

Create the public repo

  1. New GitHub repo — set to Public
  2. Name it [username]-pages or similar
  3. Initialize with a README

Enable GitHub Pages

  1. Repo → Settings → Pages
  2. Source: "Deploy from a branch" → main → / (root)
  3. Save. You'll get a URL: https://[username].github.io/[repo-name]/

Custom domain (optional)

In your DNS settings, add a CNAME: Name = your subdomain (e.g., dev), Value = [username].github.io. Then set the custom domain in repo Settings → Pages and enable HTTPS.

Add it to your profile

github:
  public_repo: "[username]/[public-repo-name]"
  public_url: "https://[your-domain-or-pages-url]/"

Keeping It Useful

Version 1.0 won't be perfect — that's fine. After a few weeks you'll notice things that are off. Update the profile when that happens. Commit the change.

  • At the end of meaningful sessions, ask Claude to commit a summary note to your context folder
  • When you start a new project, add a folder under reference/[project-name]/
  • Update the profile when your role or priorities shift — a stale profile is worse than no profile
  • Use your Slack inbox or Gmail label for quick captures throughout the day

Niles is on version 1.8 after about a year of iteration. There are deeper layers — task management workflows, multi-platform AI routing, agentic automation — that aren't in scope here. If you want to go further, ask him.
